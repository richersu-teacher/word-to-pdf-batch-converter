# 📄 word-to-pdf-batch-converter（課程計畫 Word 批次轉 PDF 工具）

這是一款專為學校「課程計畫上網公告」流程設計的 **Word 批次轉 PDF 工具**。

本工具會呼叫電腦中已安裝的 Microsoft Word 進行 PDF 匯出，轉換結果通常接近人工在 Word 中「另存成 PDF」的效果。

適合處理以下情境：

- 課程計畫檔案散落在不同年級、領域、科目或教師資料夾中
- Word 檔數量可能高達數十個甚至上百個
- 上網公告附件通常建議使用 PDF 或 ODT 等較適合公開瀏覽與保存的格式
- 手動逐一開啟 Word、另存 PDF 非常耗時，也容易漏轉或轉錯位置
- 其他行政業務若有大量 Word 文件需要公告、歸檔或轉成 PDF，也可使用本工具

本工具主要協助將既有的 Word 文件批次轉換為 PDF，減少人工重複操作。  
其中最能節省時間的情境，是像課程計畫這類「檔案多、資料夾層級多、需要大量轉檔公告」的工作。

本工具透過電腦中已安裝的 **Microsoft Word** 進行轉檔，  
因此相較於使用其他第三方轉檔工具，較能維持原本 Word 文件中的表格、圖片、頁面配置與格式。

## ✨ 核心特色

### 📄 高度保留原始版面

本工具會呼叫電腦中已安裝的 Microsoft Word 進行轉檔，  
盡可能維持 Word 文件原本的字型、表格、圖片與版面配置。

適合處理對格式要求較高的文件，例如：

- 課程計畫
- 教案
- 會議資料
- 行政表件
- 校內計畫文件
- 學校常用 Word 表格文件

---

### 📂 支援大量文件批次處理

可以一次處理多個 Word 文件，  
不需要逐一開啟 Word、另存成 PDF。

特別適合處理：

- 多個年級資料夾
- 多個領域或科目資料夾
- 多位教師分別繳交的 Word 檔
- 多層資料夾中的大量文件

---

### 🔁 只轉換 `.doc` 與 `.docx`，其他檔案依資料夾結構複製

本工具主要轉換以下 Word 文件格式：

- `.doc`
- `.docx`

其他非 Word 文件不會轉換成 PDF，  
工具會依照原本的資料夾結構複製到輸出位置。

處理過程中會略過具有以下條件的檔案：

- 隱藏檔
- 系統檔
- 常見暫存/系統雜檔（例如 `~$` 開頭、`.DS_Store`、`Thumbs.db`）

這樣可以在保留原始資料夾結構的同時，避免把不必要的系統檔、暫存檔或隱藏檔一起處理。

---

### 🛡️ 同名檔案處理

當輸出資料夾中已經有同名 PDF 或同名檔案時，工具提供：

- 覆蓋原檔
- 自動加序號另存新檔

避免重要檔案被誤覆蓋。

## ⚠️ 注意事項

使用前請先閱讀以下說明：

1. **本工具不包含 Microsoft Word**

   本工具只是協助操作電腦中已安裝的 Microsoft Word。  
   使用者需自行安裝並合法授權 Microsoft Word。

2. **未安裝 Word 無法使用**

   如果電腦沒有安裝 Microsoft Word，本工具無法獨立完成轉檔。

3. **本工具只轉換 `.doc` 與 `.docx`**

   工具只會將 `.doc` 與 `.docx` 檔案轉換為 PDF。  
   其他檔案會依照原本資料夾結構複製到輸出位置，不會轉成 PDF。

4. **轉檔結果會受電腦環境影響**

   PDF 輸出結果可能受到以下因素影響：

   - Word 版本
   - 文件使用的字型
   - 文件內的圖片、表格或特殊物件
   - 電腦系統環境

5. **特殊字型請先安裝**

   如果 Word 文件使用特殊字型，建議在轉檔電腦上也安裝相同字型，  
   以避免 PDF 版面與原始文件不同。

6. **遇到防毒或 Windows 警告時**

   若防毒軟體、Windows Defender 或校園資安系統出現提示，  
   請依照學校資訊安全規範處理。  
   若不確定是否能執行，建議洽詢學校資訊人員。

7. **建議先用少量檔案測試**

   第一次使用時，建議先用幾個測試檔案確認輸出結果，  
   確認沒有問題後，再進行大量批次轉檔。

8. **重要文件建議先備份**

   雖然工具提供同名檔案處理功能，  
   但處理大量正式文件前，仍建議先備份原始資料。

9. **轉檔期間請避免手動操作 Word**

   工具轉檔時會在背景呼叫 Microsoft Word。  
   建議轉檔期間不要同時手動開啟或操作大量 Word 文件，  
   以降低轉檔中斷或 Word 無回應的機率。

10. **大量檔案轉檔需要時間**

   若檔案數量很多，或文件內含大量圖片、表格，轉檔時間可能較長。  
   建議耐心等待工具完成處理。

## ⚠️ 不建議用於伺服器 / 自動化環境

本工具使用的是 **Microsoft Word COM Automation**。

這種方式適合在一般 Windows 桌面環境中，由使用者手動啟動工具進行轉檔。  
不建議將本工具用於以下環境：

- IIS 網站後端
- Windows Service
- 排程器長時間無人值守大量執行
- 多使用者同時呼叫的伺服器端轉檔服務
- CI/CD 或雲端自動化轉檔環境

原因是 Microsoft Office / Word 的 COM Automation 在無人值守或伺服器環境中，可能會遇到：

- Word 背景程序卡住
- 權限不足
- 視窗或對話框無法顯示
- 文件開啟後無法正常關閉
- 多使用者同時執行時互相干擾
- 轉檔流程不穩定或無法預期

本專案的定位是 **給學校行政人員或教師在本機 Windows 電腦上手動執行的桌面工具**，  
不是穩定的後端轉檔服務，也不建議作為伺服器端批次轉檔系統使用。

## 🛠️ 開發說明

本工具使用 C# WinForms 開發，  
介面設計以簡單、穩定、容易操作為主。

轉檔時會呼叫使用者電腦中已安裝的 Microsoft Word，  
由 Word 本身負責將文件輸出為 PDF，  
因此較能保留原始 Word 文件的排版效果。

本工具不使用第三方排版引擎重新解析 Word 文件，  
主要目標是降低批次轉檔時的人工操作負擔，  
並盡可能維持原本文件在 Microsoft Word 中的顯示效果。

## 🧑‍💻 技術說明（給開發者）

本專案為 Windows 桌面工具，主要技術如下：

- C#
- WinForms
- .NET 10
- Microsoft Word COM Automation
- `dynamic` / Late Binding
- `Type.GetTypeFromProgID("Word.Application")`

本工具沒有直接綁定特定版本的 Office Interop DLL，  
而是透過 Late Binding 動態呼叫本機已安裝的 Microsoft Word。

這樣做的目的，是降低不同電腦、不同 Office / Word 版本之間的相容性問題。

核心轉檔概念：

1. 掃描使用者指定的來源資料夾
2. 略過隱藏檔、系統檔與常見暫存雜檔
3. 遇到 `.doc` 或 `.docx` 時，呼叫本機 Microsoft Word 開啟文件
4. 由 Microsoft Word 匯出 PDF
5. 其他非 Word 檔案依原資料夾結構複製到輸出位置
6. 遇到同名檔案時，依使用者選擇覆蓋或自動加序號

開發者需要注意：

- 本專案依賴 Windows 桌面環境
- 本專案依賴使用者本機安裝的 Microsoft Word
- 不適合移植成 Linux / macOS 工具
- 不適合直接改造成 Web API 或伺服器端轉檔服務
- 若要開發伺服器端文件轉檔服務，應改用其他適合伺服器環境的文件處理方案
